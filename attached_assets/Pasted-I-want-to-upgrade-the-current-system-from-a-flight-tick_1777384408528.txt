I want to upgrade the current system from a flight-ticket booking system only into a full internal employee CRM + ticket management system for a travel agency.

Main goal:
Each employee in the company should be able to manage customers, add customer details, follow up with them, track previous tickets, add notes, and see all related data in the main dashboard.

Important rules:
- No dummy data.
- No fake buttons.
- Every button must have a real route and a real API action.
- Every form must save data correctly.
- All pages must be connected properly.
- Keep the existing booking/search functionality, but expand the system around it.

Required new modules/pages:

1. Customers Management
Create a full Customers page where employees can:
- Add a new customer
- Edit customer data
- Delete customer if allowed
- View customer profile
- Search/filter customers

Customer fields:
- Full name
- Phone number
- WhatsApp number
- Email
- Nationality
- Passport number
- National ID
- Address
- Source: Facebook / WhatsApp / Walk-in / Referral / Other
- Assigned employee
- Customer status: New / Interested / Follow-up / Booked / Cancelled / Lost
- Created date
- Last contacted date

2. Customer Profile Page
Each customer must have a detailed profile page containing:
- Customer basic information
- All previous tickets/bookings
- All follow-up notes
- Upcoming reminders
- Customer status history
- Quick actions:
  - Add ticket
  - Add note
  - Add reminder
  - Change status
  - Call/WhatsApp link if phone exists

3. Customer Notes & Follow-up System
Create a notes system for each customer.

Each note should include:
- Note text
- Created date
- Created by employee
- Follow-up date
- Follow-up status: Pending / Done / Missed
- Related ticket if available

Example:
If the customer said yesterday: “I will buy on Saturday”, the employee should add a note with Saturday as the follow-up date.
On Saturday, the employee contacts the customer and adds another note saying what happened.

The system should allow:
- Add daily notes
- View notes timeline
- Filter notes by date/status
- Mark follow-up as done
- Show today’s follow-ups in the dashboard

  - Unpaid

- Baggage details
- Employee responsible
- Notes
- Created date

Each ticket should have its own page with:
- Ticket full details

- Payment status

- Actions:
